Advanced Hardware Mod & Firmware Re-Engineering

Role: Systems Engineer / Hardware Modder

Tools: UEFI/Flashrom Linux Gemini AI (Diagnostics) Hardware modifications

Project Overview

This project involved modernizing "end-of-life" Chromebook hardware by bypassing proprietary, hardened security ecosystems. Unlike standard OS migrations, this required decoupling Google's "Read-Only" firmware and physical write-protection (WP) locks to transform a restricted device into a high-performance Linux workstation.

Among the projects summarized in this portfolio, this one is probably least applicable to my current or future job-related responsibilties. However, it's a project I'm extremely proud of because it was an ultimately successful attempt on my part to take on a project in an area where I am not comfortable. It was also somewhat risky in the sense that I could have "bricked" one of my own laptops if I failed. I'm also especially proud of this project because it represents how the use of AI can allow a person to venture into an unfamiliar domain and not only learn something new, but find success.

Technical Implementation

The workflow required a multi-layered approach, transitioning from physical hardware deconstruction to low-level software debugging and kernel configuration.

[Status] Polling hardware registers...
[Command] crossystem wpsw_cur
[Output] WP: enabled (Register Latch Detected)
[Action] Initializing flashrom UEFI write...

Phase 1: Hardware Deconstruction & WP Removal

Performed precision disassembly to remove the physical Write-Protect screw. This stage involved identifying the specific motherboard layout and managing the Embedded Controller (EC) state to prep for firmware modification.

Phase 2: Firmware & EC Debugging

Utilized the Linux shell and utilities like flashrom and crossystem to poll motherboard status registers. This phase focused on overcoming "locked" states that persisted even after physical modifications.

Phase 3: OS Migration

Successfully flashed custom UEFI firmware (MrChromebox) and transitioned the system to a lightweight, open-source Linux operating system. (Which I have since updated a second time.)

Key Challenge: The "Ghost Connection"

The Hurdle: Despite removing the physical Write-Protect screw, the system continued to report "WP: enabled," preventing the firmware flash. Standard documentation offered no solution for this persistent software/hardware mismatch.

The Fix: By using Gemini as a real-time diagnostic partner, I synthesized documentation on EC latches and kernel-level reporting. This led to the discovery of microscopic metallic debris bridging the contact pads on the motherboard. A precision cleaning of the contact pads cleared the "ghost connection," allowing for a successful UEFI image write.

Final Result: Successfully converted a restricted, proprietary device into a fully customizable, high-performance Linux workstation, establishing a replicable methodology for legacy hardware optimization.
